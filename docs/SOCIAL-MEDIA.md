## LinkedIn Post

### Post

> **We built an AI memory system that actually remembers.**
>
> Every AI session starts from scratch. Chat with an assistant today, and tomorrow it's forgotten your name, your preferences, your entire project context.
>
> The industry keeps building bigger context windows. We think that's the wrong approach.
>
> **Rik Context Engine** is an open-source Python framework that gives AI agents a persistent, hierarchical memory system modeled after human cognition:
>
> 📦 **Episodic Memory** — Session-level, high-fidelity snapshots
> 🧠 **Semantic Memory** — Long-term structured knowledge (subject→predicate→object)
> ⚙️ **Procedural Memory** — Skills, workflows, and how-to knowledge
>
> **What makes it different:**
>
> ✅ Auto-importance scoring — every message scored on 4 weighted dimensions; low-importance content pruned first
> ✅ Coherence validation — ensures pruned context remains logically intact
> ✅ Knowledge graph — entity relationships + semantic vector search
> ✅ Task decomposition — goal → dependency-respecting task graph
> ✅ Self-improvement loop — reflection analyzer learns from mistakes
>
> **Stack:** Python + SQLite + (optional) ChromaDB/Ollama
>
> No external database required. Local-first. AGPL-3.0.
>
